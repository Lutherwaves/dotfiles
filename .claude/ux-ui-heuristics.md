# UX â€“ UI Heuristics (Nielsen's 10)

_Use when designing screens, components, flows._

## 1. Visibility of System Status

- Show loading states, skeletons, progress on async actions.

## 2. Match with Real World

- Use domain-appropriate language; avoid internal jargon.

## 3. User Control & Freedom

- Add cancel/undo where possible; non-destructive defaults.

## 4. Consistency & Standards

- Consistent layout, naming, icons; follow design system.

## 5. Error Prevention

- Disable invalid actions; validate before submit.

## 6. Recognition vs Recall

- Visible filters, hints, contextual labels (no hidden modes).

## 7. Flexibility & Efficiency

- Keyboard shortcuts for power users; simple defaults for novices.

## 8. Minimalist Design

- Avoid unnecessary elements; clear visual hierarchy.

## 9. Help Recover from Errors

- Clear error messages + next steps.

## 10. Help & Documentation

- Inline help, tooltips, links to docs.
